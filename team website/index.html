About Evalyo

The Problem

Students at Hunter College use the site RateMyProfessors to evaluate and choose their professors. RateMyProfessors is wdiely used application across campuses in the United States, where students can review former professors. Students both generate and recieve the site's content.

However, RateMyProfessors has loose community standards which often leads to site abuse by disgruntled students. The site is filled with devastating reviews, and in return professors avoid the site.

A possible solution would be to use course evaluations presented by Hunter. Professors often encourage students course evaluations, and a great percentage of students completes these evaluation surverys. Hunter student could rely on the information from this database instead of RateMyProfessors. This database does provide more evaluation data than RateMyProfessors but without the harsh words.

Sadly, the problem with this database is the way information is displayed:

  • No option to aggregate course evalutions.
  • No option to query based by course or professor.
  • Not a friendly interface.

The Solution

Enter Evalyo! We will use modern web frameworks to improve the current site by adding the following:

    Professor/Course Base Querying

    Instead of recieveing a list of courses with a professor that matches your result, queries will aggregate the course to a single professor. Another feature would show when a professor whcih semester a professor teacher a course.

    Hitorical Aggregation

    Student will be able to see a professor rating across multiple semesters, the number will be in a total average that weights recent semesters more heavily.

    Class Based Querying

    Unlike RateMyProfessors or the current Hunter based system, students could search based on courses and see result of all professors who have taught the course.

    Student Friendly Design

    Current data presents information with an obtuse language. The app would reduce metrics from a total of 20+ to 5 key metrics.
